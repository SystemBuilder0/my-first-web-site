# voice-typer

F2를 누르면 녹음 시작, 다시 누르면 종료 → GPU에서 Whisper(turbo)로 텍스트 변환 →
커서가 있는 곳에 자동 타이핑 + 항상 떠 있는 안전창(overlay)에도 동일하게 기록.

> **중요 (정직한 고지):** 이 코드는 GPU/마이크/디스플레이가 없는 원격 리눅스
> 샌드박스에서 작성됐다. `faster-whisper`의 API(모델 이름 "turbo" 존재 여부,
> `transcribe()`가 numpy 배열을 받는지 등)는 실제로 라이브러리를 설치해서
> 소스코드로 직접 확인했고, 문법 오류도 `py_compile`로 확인했다. 하지만
> **실제 녹음, GPU 추론, 전역 단축키(F2) 동작은 이 환경에서 실행해볼 수 없어서
> 검증하지 못했다.** 이후 상범님의 실제 PC(RTX 3060, Antelope ZenGo SC 오디오
> 인터페이스)에서 여러 차례 실행하며 발견된 문제들(오디오 라이브러리 크래시,
> GPU DLL 인식 실패, 관리자 권한 계정 꼬임 등)은 아래 내용에 전부 반영해서
> 고쳤고, 실제로 동작하는 것까지 확인했다.

## 요구 사항

- Windows 10/11
- NVIDIA GPU, VRAM 8GB 이상 (RTX 3060 이상), 최신 그래픽 드라이버
- Python 3.10 또는 3.11 (Windows에 `py` 런처로 설치돼 있어야 함)
- 마이크 (여러 채널을 가진 오디오 인터페이스도 지원, 아래 참고)

## 처음 한 번만: 설치

1. `voice-typer` 폴더 안의 **`setup_voice_typer.bat`**을 더블클릭한다.
   (관리자 권한으로 실행하지 않는다 — 그냥 더블클릭)
2. 검은 창이 뜨고 필요한 패키지들을 자동으로 설치한다. 처음이라면 몇 분
   걸릴 수 있다.
3. "Setup finished."라는 문구가 뜨면 완료. 아무 키나 눌러 창을 닫는다.

이 과정은 **딱 한 번만** 하면 된다. 이후에는 아래 "실행"만 반복하면 된다.

## 그 다음부터: 실행

`voice-typer` 폴더 안의 **`run_voice_typer.bat`**을 더블클릭한다. 이것만
하면 된다.

- 매번 명령 프롬프트를 열거나 폴더를 `cd`로 이동할 필요 없음.
- 자주 쓴다면 이 파일의 바로 가기를 만들어 바탕화면에 둬도 된다
  (오른쪽 클릭 → "바로 가기 만들기" → 바탕화면으로 옮기기).
- **바로 가기 속성에서 "관리자 권한으로 실행"에 체크하지 않는다.**
  (이유는 아래 "왜 관리자 권한으로 실행하면 안 되나" 참고)

실행하면:
1. "voice-typer 안전창"이라는 항상-위-창이 뜬다.
2. 백그라운드에서 Whisper turbo 모델을 GPU에 로딩한다 (첫 실행은 모델을
   허깅페이스에서 다운로드하므로 인터넷 필요, 이후엔 캐시돼서 빠름).
3. 안전창에 "[모델 로딩 완료]"가 뜨면 준비 완료.
4. 아무 곳에나(메모장, 브라우저, Cursor 등) 커서를 두고 F2를 눌러 말하고,
   다시 F2를 눌러 종료.
5. 잠시 후 커서가 있던 곳에 텍스트가 타이핑되고, 동시에 안전창에도 같은
   텍스트가 시각과 함께 남는다.

프로그램이 에러로 죽어도 창이 바로 닫히지 않고 "계속하려면 아무 키나
누르십시오..." 상태로 남아있으니 에러 메시지를 확인할 수 있다.

## 왜 관리자 권한으로 실행하면 안 되나

이 PC에는 윈도우 계정이 두 개 있다(junhy, jayjh). **"관리자 권한으로
실행"을 선택하면 자동으로 jayjh 계정으로 전환되고, 그 계정에는 별도의
Python이 설치돼 있어서** `setup_voice_typer.bat`으로 설치한 패키지를
찾지 못하는 "분명히 설치했는데 없다고 나온다"는 혼란이 반복해서 생겼다.

일반 권한(그냥 더블클릭)으로 실행해도 메모장, 브라우저, Cursor 같은 보통
프로그램에 타이핑하는 데는 전혀 문제가 없다. 유일한 제약은, **관리자
권한으로 떠 있는 창(예: 관리자 권한 명령 프롬프트)에 포커스가 가 있을
때는 F2가 눌러도 인식되지 않는다**는 것뿐이다. 평소 쓰는 데는 지장이
없으므로, 특별한 이유가 없다면 **항상 그냥 더블클릭**해서 쓰면 된다.

## GPU가 인식되지 않을 때 (문제 해결용 — 보통은 신경 쓸 필요 없음)

`voice_typer.py`는 시작할 때 `nvidia-cublas-cu12`, `nvidia-cudnn-cu12`
패키지가 설치된 위치를 스스로 찾아서 `os.add_dll_directory`와
**환경변수 PATH 양쪽 모두에** 등록한다. ctranslate2가 `cublas64_12.dll`을
런타임에 `LoadLibrary`로 불러오는데, 이 방식은 `add_dll_directory`만으로는
못 찾고 PATH에 있어야만 찾기 때문이다. 이 처리 덕분에 **PATH를 수동으로
설정하거나 환경변수를 직접 편집할 필요가 없다.**

그래도 GPU 인식 에러(`cublas64_12.dll` 관련 메시지 등)가 계속 난다면:

1. `setup_voice_typer.bat`을 다시 실행해서 두 패키지가 실제로 설치됐는지
   확인한다.
2. 그래도 안 되면 `config.py`에서 `DEVICE = "cpu"`, `COMPUTE_TYPE = "int8"`로
   바꿔서 CPU로도 일단 동작하는지 확인한다 (체감 속도는 많이 느려짐 —
   GPU 문제 진단용으로만 사용).

## 여러 채널을 가진 오디오 인터페이스(ZenGo SC 등) 쓸 때

오디오 입력은 `sounddevice`(PortAudio)가 아니라 `soundcard`(WASAPI)를 쓴다.
PortAudio는 초기화할 때 모든 오디오 백엔드(WDM-KS 포함)의 장치를 훑는데, 일부
다채널 인터페이스(Antelope ZenGo SC 등) 드라이버에서 이 과정 자체가 프로세스를
통째로 죽여버리는 게 실제로 확인됐다 (에러 메시지도 없이 그냥 종료됨).
Premiere Pro 같은 프로그램은 WASAPI/ASIO로 접근해서 문제없이 녹음되는 것과
같은 이유로, `soundcard`(WASAPI 전용)로 바꾸면 이 문제를 피해간다.

`config.py`의 `MIC_NAME`에 원하는 마이크 이름의 일부를 적으면 된다:

```python
MIC_NAME = "ZenGo"
```

실행하면 안전창에 `[마이크 선택] <실제로 골라진 장치 전체 이름>`이 찍힌다.
ZenGo SC처럼 채널이 여러 쌍(1/2, 3/4, 5/6, 7/8)인 인터페이스는 이름이 겹쳐서
엉뚱한 채널이 골라질 수 있다. 그럴 땐 안전창에 찍힌 전체 이름을 그대로
`MIC_NAME`에 붙여넣어 정확히 지정하면 된다 (예: `"ZenGo SC USB Audio Driver Recording 1/2"`).
`MIC_NAME = None`으로 두면 윈도우 기본 마이크를 그대로 쓴다.

## 설정 바꾸기 (`config.py`)

| 값 | 설명 |
|---|---|
| `HOTKEY` | 단축키 (기본 `"f2"`) |
| `MODE` | `"toggle"`(F2로 수동 시작/종료, 기본) 또는 `"vad"`(F2로 상시감지 모드 on/off, 그 안에서 소리크기로 자동 문장 구분) |
| `MODEL_SIZE` | `"turbo"`(기본, 빠름) / `"large-v3"`(정확도 우선) / `"medium"`, `"small"`(VRAM 부족 시) |
| `DEVICE`, `COMPUTE_TYPE` | GPU: `"cuda"` + `"float16"`. CPU 폴백: `"cpu"` + `"int8"` |
| `LANGUAGE` | `"ko"` 고정. 자동감지 원하면 `None` |
| `VAD_DB_THRESHOLD` | `MODE="vad"`일 때 말하는 중으로 판단하는 dB 기준. 값이 부정확하면 너무 자주/안 끊기니 실사용하면서 조정 |
| `VAD_SILENCE_MS` | 이만큼(ms) 조용하면 한 문장 끝으로 보고 바로 변환 시작 |
| `TYPE_AT_CURSOR` | `False`로 하면 실제 타이핑은 안 하고 안전창에만 기록 |
| `MIC_NAME` | 사용할 마이크 이름의 일부 (예: `"ZenGo"`). `None`이면 윈도우 기본 마이크 사용 |

## 처음 실행할 때 확인 순서 (문제가 생기면 이 순서로 원인을 좁힐 것)

1. `run_voice_typer.bat` 실행 → 안전창이 뜨는지 확인 (안 뜨면 tkinter/디스플레이
   문제).
2. "[모델 로딩 완료]"가 안전창에 뜨는지 확인 (안 뜨면 GPU/faster-whisper 설치
   문제 — 검은 창에 찍히는 에러 메시지 확인, 위 "GPU가 인식되지 않을 때" 참고).
3. 메모장 등 텍스트 입력창에 커서를 두고 F2 → 몇 초 말하고 → F2.
   안전창에 "[녹음 시작]" → "[녹음 종료 - 변환 중...]" → 인식된 텍스트가
   순서대로 뜨는지 확인.
4. 메모장에도 같은 텍스트가 타이핑됐는지 확인. 안 됐다면 지금 포커스가
   관리자 권한 창에 가 있는 건 아닌지 먼저 확인한다 (일반 창으로는
   보통 잘 된다 — 위 "왜 관리자 권한으로 실행하면 안 되나" 참고).

## 알려진 제약

- `keyboard.write()`는 유니코드(한글 포함)를 Windows의 `SendInput`
  유니코드 모드로 전송하므로 원칙적으로 한글도 그대로 입력되지만, 일부
  게임처럼 `SendInput`을 무시하고 raw 입력만 받는 특수 프로그램에서는
  타이핑이 안 들어갈 수 있다. 이런 경우에도 안전창에는 항상 남는다.
- 관리자 권한으로 떠 있는 창에 포커스가 가 있으면 F2가 잡히지 않는다
  (일반 사용에서는 문제되지 않음).
- 상시 감지(`MODE="vad"`) 모드의 dB 임계값은 마이크/방 소음에 따라 달라서
  기본값이 상범님 환경에 맞다는 보장은 없다. 처음엔 `MODE="toggle"`로
  검증한 뒤 `"vad"`로 넘어가는 것을 권장.
- 모델을 처음 로딩할 때 허깅페이스에서 수 GB를 내려받는다 (turbo 모델은
  약 1.6GB 안팎). 이후에는 로컬 캐시(`~/.cache/huggingface`)에서 불러온다.
